# Angular Storefront — Small Mental Model Guide

# The Big Picture

This entire project teaches students how modern applications grow from:

```txt
simple frontend
        ↓
interactive frontend
        ↓
structured frontend
        ↓
API-driven frontend
        ↓
full stack application
        ↓
database-backed system
```

---

# The Core Mental Model

Think of the project like building a real business.

---

# Stage 1 — The Store Building

```txt
HTML + CSS
```

Students build the visual store.

Learning:

- layout
- sections
- cards
- buttons
- responsive UI

Mental model:

> Build the shell first.

---

# Stage 2 — Angular Controls the Store

```txt
Angular templates
```

The UI becomes dynamic.

Learning:

- interpolation
- bindings
- Angular components

Mental model:

> Angular connects data to the UI.

---

# Stage 3 — The Store Reacts Automatically

```txt
Signals
Computed
Reactive state
```

Learning:

- search
- cart updates
- totals
- reactive UI

Mental model:

> Signals are reactive memory.

---

# Stage 4 — The Store Feels Professional

```txt
UX improvements
```

Learning:

- messages
- disabled states
- feedback
- user experience

Mental model:

> Good apps communicate clearly.

---

# Stage 5 — The Store Becomes Modular

```txt
Components
```

Learning:

- reusable UI
- Input/Output
- separation of concerns

Mental model:

> Big apps are built from smaller pieces.

---

# Level 4 — Shared Logic

```txt
Services
```

Learning:

- centralized logic

Mental model:

# Final Architecture Mental Model

```txt
Angular Frontend
        ↓
Express Backend API

MongoDB Database
```

---

# Final Student Takeaway

Students are not just learning Angular.

They are learning:

```txt
how real applications evolve
how software scales
how frontend and backend communicate
how data flows through systems
```

The biggest lesson:

> Every new stage solves a problem created by the previous stage.

That is real software engineering thinking.
